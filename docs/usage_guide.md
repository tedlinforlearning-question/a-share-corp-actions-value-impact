# 使用指南

## 一、启动 Skill

在 WorkBuddy/CodeBuddy 对话中，直接描述你的任务即可触发 Skill。例如：

```
"帮我做某公司的IPO估值分析，招股书在 D:\reports\招股书.pdf，审计报告在 D:\reports\审计报告.pdf"
```

Skill 会自动：
1. 判断项目类型（IPO / M&A置入 / M&A出售 / 再融资-纯补流 / 再融资-补流+募投 / 再融资-收购+配套）
2. 检查报告完备性
3. 提取估值基准日
4. 执行对应的工作流

---

## 二、IPO 估值工作流

### Step 1: 提供报告文件

确保以下文件在本地可访问路径：
- 招股说明书（申报稿/注册稿）`.pdf`
- 审计报告（申报稿）`.pdf`

### Step 2: 前置检查

Skill 会自动执行：
- **报告完备性检查**：确认文件都存在且可读
- **在建工程分析**（0A节）：提取在建工程明细，与募投项目交叉比对
- **时间距离检查**：如果基准日距今 >6个月，提示获取最新行业数据

### Step 3: 数据提取

Skill 会自动从 PDF 中提取：
- 合并资产负债表、利润表、现金流量表
- 折旧摊销四项（固定资产折旧、无形资产摊销、长摊摊销、使用权折旧）
- CAPEX（购建固定资产支付的现金）
- 募投项目参数（总投资、建设期、达产收入、IRR）

### Step 4: 可比公司 WACC

调用 Wind（或 AKShare/Tushare）获取可比公司数据：
- β值（3年月频）
- 总市值、有息负债
- 有效税率

计算 βU 去杠杆矩阵，得到目标 βU。

### Step 5: 独立 DCF

基于现有业务构建独立 DCF 模型：
- 收入增长率：参考行业增速或历史 CAGR
- EBIT 利润率：最近2年平均
- CAPEX：实际历史值 + 在建工程修正
- 产出：**不含募投的每股价值**

### Step 6: 募投项目 FCFF

基于在建工程分析结果构建募投项目增量 FCFF：
- 爬坡节奏：根据在建工程进度修正（非招股书原假设）
- CAPEX：仅计入剩余未投入部分
- 产出：**募投项目增量价值**

### Step 7: 备考合并 DCF

合并独立 DCF 和募投 FCFF：
```
合并FCFF = 独立FCFF + 募投FCFF
合并股权价值 = DCF(合并FCFF, 合并WACC)
合并每股价值 = 合并股权价值 / (基准日股本 + 发行数量上限)
```

### Step 8: 敏感性分析

生成 WACC × g 二维敏感性矩阵。

### Step 9: 假设清单输出

Skill 会自动输出关键假设清单和敏感性提示。

## 三、M&A 并购置入工作流

### Step 1: 提供报告文件

确保以下文件在本地可访问路径：
- 上市公司近3年年报（每年一份）
- 重组报告书/重大资产购买报告书
- 标的公司审计报告/模拟报表
- （可选）评估报告/估值报告

### Step 2: 前置检查

Skill 会自动执行：
- **报告完备性检查**
- **时间距离检查**：如果基准日距今 >6个月，提示获取最新行业数据

### Step 3-8: 建模流程

与 IPO 类似，主要差异：
- **独立 DCF**：基于上市公司历史财务数据
- **标的 FCFF**：基于标的公司审计报告 + 协同效应
- **合并 WACC**：需迭代求解（D/E变化）
- **股份处理**：发股购买 → 合并股数 = 基准日 + 发行上限；现金收购 → 股数不变

## 四、M&A 重大资产出售工作流

### Step 1: 提供报告文件

确保以下文件在本地可访问路径：
- 上市公司近3年年报
- 重组报告书（含"重大资产出售"）
- 置出资产专项审计报告
- 备考审阅报告
- 资产评估报告

### Step 2: 出售类型判定

Skill 首先判断标的是否适合 DCF 建模：
- **并表子公司出售** → ✅ DCF 适用
- **金融资产/联营企业出售** → ❌ DCF 不适用，仅做资产桥接

### Step 3: 备考扣非分离

从备考合并利润表提取数据，分离经常性/非经常性损益：
- 经营性利润（营业收入 - 营业成本 - 税金 - 三费）
- 非经常性投资收益（分离处置收益/一次性收益）
- **扣非归母净利润作为 DCF 起点**

### Step 4-7: 剩余业务 DCF

- **出售前 DCF 判定**：如果归母权益<0或持续巨亏，出售前DCF不适用
- **剩余业务 WACC**：沿用重组报告书参数，微调 D/E 和 Rc
- **剩余业务 DCF**：扣非口径 FCFF，投资收益=0
- **备考对比**：出售前后每股净资产 + 每股DCF + 每股收益对比

## 五、再融资-纯补流 工作流

### Step 1: 提供报告文件

确保以下文件在本地可访问路径：
- 募集说明书（注册稿/申报稿/修订稿）
- 审计报告（或上市公司年报全文PDF，用于提取BPS/资产负债率等）

### Step 2: 参数提取

Skill 自动从募集说明书和审计报告提取：
- 发行方案：发行数量上限、发行价格/底价、定价基准日、募集资金总额
- 财务数据：归母净资产、每股净资产(BPS)、资产负债率、有息负债
- 资金缺口：最低现金保有量、未来三年营运资金缺口
- 利息支出、平均借款利率

### Step 3: 摊薄分析（7 Sheet Excel）

自动生成完整摊薄分析底稿：
- **Sheet 1 参数汇总**：发行方案 + 核心财务数据
- **Sheet 2 持股比例摊薄**：股本结构变化 + 稀释率
- **Sheet 3 BPS变化**：每股净资产增厚/摊薄判断（发行价 > BPS前 → 增厚）
- **Sheet 4 EPS摊薄**：悲观/基准/乐观三情景
- **Sheet 5 利息节约**：补流→偿还借款→利息节约→利润改善
- **Sheet 6 Pro Forma**：发行前后资产负债表对比
- **Sheet 7 综合对比**：全维度汇总 + 最终评判

### Step 4: 最终判断

核心结论：
- 发行价是否低于BPS（增厚还是摊薄老股东？）
- 偿还借款后的杠杆改善幅度
- 每股价值综合影响

> **本场景不需要 DCF**：纯补流没有具体投资标的，不涉及项目估值。

## 六、再融资-补流+募投 工作流

### Step 1: 提供报告文件

- 募集说明书（含募投项目效益测算表）
- 审计报告（或年报全文PDF）

### Step 2: 参数提取

除纯补流参数外，额外提取：
- 募投项目效益测算（10年预测期：收入、成本、利润、IRR、回收期）
- 建设期、达产进度、总投资额

### Step 3: 摊薄分析 + 项目DCF（10 Sheet Excel）

在纯补流 7 Sheet 基础上追加 3个 Sheet：
- **Sheet 8 项目一DCF**：利润预测→FCF→NPV
- **Sheet 9 项目二DCF**：同上
- **Sheet 10 合并估值**：存量P/E + Σ项目NPV = 发行后内在价值

### Step 4: 最终判断

```
发行后每股价值 = (存量估值 + Σ项目NPV) / 发行后股本
净效应 = 发行后每股 - 发行前每股

IF 净效应 > 0: 项目价值创造超过股本稀释 ✓
ELIF 净效应 ≈ 0: 项目开辟新赛道但短期不增每股价值
ELSE: 股本稀释过大 ⚠
```

> **特别注意**：主板再融资发行底价 ≥ 经审计BPS。

## 七、再融资-收购+配套 工作流

> 最复杂场景：定增募集资金→现金收购标的公司股权。再融资与M&A的交叉地带。

### Step 1: 提供报告文件

- 募集说明书（含收购方案+发行方案）
- 标的公司审计报告或模拟报表
- 上市公司年报或审计报告

### Step 2: 参数提取

除发行方案外，额外提取：
- 标的公司近2年财务数据（收入/成本/利润/资产负债）
- 标的评估值（资产基础法/收益法）、P/E/P/B乘数
- 发行人与标的的关联关系及内部采购金额（备考合并关键）

### Step 3: 多维度分析（7 Sheet Excel）

- **Sheet 1 交易概览**：交易结构 + 发行方案
- **Sheet 2 摊薄分析**：持股比例 + BPS变化
- **Sheet 3 标的财务分析**：2年财务 + 估值对标
- **Sheet 4 标的DCF**：收入预测→EBIT→NOPAT→FCF→股权价值
- **Sheet 5 备考合并**：关联交易抵消（收入+成本双重抵消）
- **Sheet 6 协同效应**：成本节约量化→协同永续价值
- **Sheet 7 价值影响**：稀释效应 vs 标的价值 vs 协同价值 → 净效应

### Step 4: 核心难点

- **关联交易抵消**：标的→发行人的内部销售在合并时双向抵消
- **协同量化**：协同永续价值 / 收购对价 = ? （>100%→极度划算）
- **综合判断**：标的DCF+协同能否覆盖股本稀释？

## 八、常见问题

### Q: 审计报告是扫描版怎么办？

A: Skill 会自动检测。如果 PyMuPDF 提取为空，会尝试从招股书中提取财务数据作为替代。如确实需要 OCR，可安装 easyocr 或 paddleocr。

### Q: 没有 Wind 终端怎么办？

A: 可以使用 AKShare 作为免费替代方案，支持 β 值和可比公司财务数据获取。安装：`pip install akshare`。

### Q: 敏感性分析中 WACC 范围如何确定？

A: 默认 WACC 7%-13%，g 1%-3%。这些范围覆盖了大多数 A 股上市公司的合理区间。

### Q: 如何自定义假设参数？

A: 在建模过程中，Skill 会询问每个关键假设的取值。你也可以直接指定，例如："收入增速用10%递减至3%，EBIT利润率用18%"

### Q: 输出 Excel 的样式可以自定义吗？

A: Excel 样式遵循投行实务规范（见 SKILL.md 第13节）。如需调整，可在 SKILL.md 中修改样式规范。
